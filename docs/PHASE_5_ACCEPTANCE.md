# Phase 5 验收记录

日期：2026-07-18

## 已完成

- 模考模板以配置保存题量、组卷模式、时长和单题权重；页面不写死官方未
  公布的题量或单题分值。
- 支持固定卷和随机组卷，只选择最新版本、已发布且可自动判分的客观题，
  `disputed` 等非发布题目不会进入模考。
- 服务端保存 `started_at` 和 `deadline_at`，刷新读取不重置截止时间；
  到期读取会话或客户端倒计时归零会自动交卷。
- 每次选择答案立即写入服务端；断网前先写浏览器本地队列，联网后自动
  重试并只清除已经确认同步的答案版本。
- 支持题号导航、已答状态、标记、暂离恢复、提前交卷和幂等重复交卷。
- 交卷前的会话响应不包含正确答案或解析；交卷后才提供逐题得分、答案和
  解析。
- 客观题采用既有精确匹配规则判分，结果按章节和题型拆解；错题和未作答
  题进入既有错题本与复习队列。
- 生产环境缺少Supabase时继续失败关闭，本地开发使用进程内仓库。
- 手机模考界面提供集中倒计时、保存状态、答题卡和底部上一题/下一题
  操作；参考用户截图的信息层级，但未复制第三方品牌、图片或题库内容。

## 官方规则复核

- 2026年上半年官方批次安排确认：初、中级基础知识与应用技术两科连考，
  总作答时长240分钟；基础知识最短90分钟、最长120分钟，节余时间可供
  第二科使用。本资格安排三个批次，个人实际批次以准考证为准。
- 官方资格说明确认：基础知识为机考选择题，应用技术（案例分析）为机考
  问答题。
- 人社部相对固定合格标准确认：计算机技术与软件资格考试各科合格标准为
  试卷满分的60%，特殊调整以另行通知为准。
- 当前官方公开页面没有给出题量、单题分值或每科满分，因此这些值没有被
  当作官方事实写死。来源URL见 `docs/CONTENT_SOURCE_AUDIT.md`。

## 尚待外部环境验收

当前电脑没有Supabase CLI、PostgreSQL客户端、Docker和目标Supabase云
项目，因此无法真实执行Phase 5迁移和RPC。正式部署前必须：

1. 在已完成前四份迁移的干净Supabase项目中执行
   `202607180005_phase5_mock_exam.sql`。
2. 用固定卷和随机卷分别验证题量不足时回滚、题目版本快照和争议题排除。
3. 验证刷新、跨设备读取和服务端超时交卷不会改变原 `deadline_at`。
4. 模拟断网、重连和重复请求，验证答案最终一致且重复交卷不重复写作答。
5. 用两个测试 `user_id` 验证模板读取、会话、答案、成绩和错题状态不串
   数据。
6. 分别以 `anon`、`authenticated` 和 `service_role` 验证表与RPC权限。
7. 部署后在真实手机上完成长时倒计时、后台切换、断网恢复和交卷验收。

完成这些外部检查前，Phase 5本地代码验收通过，但云端数据库和正式部署
不能标记为完成。
