# Phase 2 验收记录

日期：2026-07-18

## 已完成

- 固定顺序的18个一级考试模块。
- 支持模块、章节、节三级目录，空模块显示真实空状态。
- 仅展示已发布知识点，支持标题、摘要、正文和目录路径搜索。
- 知识点详情展示摘要、核心内容、考试关注点、易混淆点、工作示例、
  公式、重要度、难度、内容版本、来源与版权。
- 个人笔记、掌握状态和收藏按固定所有者 `user_id` 隔离保存。
- JSON导入使用Zod校验文件结构、字段长度、目录层级、来源、版权、
  审核状态、危险标记和重复ID。
- 导入前展示新增、更新、跳过和拒绝项；存在拒绝项时不能提交。
- 导入按 `external_id` 和版本幂等处理；相同版本跳过，低版本拒绝。
- Supabase RPC在单个事务中导入目录和知识点；异常时回滚内容变更并
  保留失败任务报告。
- 数据表启用RLS，浏览器角色无表权限，导入RPC只授予
  `service_role`。
- 内置知识卡片和E2E导入内容均为原创示例，没有复制商业教材或题库。

## 尚待外部环境验收

当前电脑没有Supabase CLI、PostgreSQL客户端和目标Supabase云项目，
因此无法在本机真实执行数据库迁移。正式部署前必须：

1. 创建Supabase云项目并配置服务端环境变量。
2. 在空数据库依次执行Phase 1和Phase 2迁移。
3. 核对18个模块顺序、三级目录种子和原创知识卡片。
4. 用合法、重复、低版本和故意失败文件验证事务提交、回滚与导入报告。
5. 用两个测试 `user_id` 验证个人笔记和状态不会串数据。
6. 分别使用 `anon`、`authenticated` 和 `service_role` 验证表与RPC权限。
7. 部署后用真实手机完成目录、笔记、导入和搜索验收。

这部分完成前，Phase 2本地代码验收可以通过，但云端数据库和正式部署
不能标记为完成。
