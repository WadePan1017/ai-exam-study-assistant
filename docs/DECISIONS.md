# 项目决策记录

## 2026-07-18：第一版范围

- 产品只供一位用户备考系统集成项目管理工程师（中级）。
- 目标是先在手机上稳定学习和刷题，不建设多用户账号体系。
- 第一版仍保留 `user_id` 数据边界，避免未来增加登录时重做数据模型。

## 2026-07-18：访问与权限

- 使用固定 `owner` 身份。
- `owner` 同时拥有学习和内容管理能力。
- 不使用邮箱注册或 Supabase Auth。
- 生产网站使用服务端访问口令和30天 HttpOnly Cookie。
- 浏览器不得直接持有 Supabase service role key。

## 2026-07-18：环境

- 开发环境在本机运行。
- 正式环境使用一个 Supabase 云端项目。
- 第一版不单独建设 preview/staging 环境。
- 手机通过部署后的 Web/PWA 地址访问。

## 2026-07-18：考试配置

- 科目一：系统集成项目管理基础知识，机考选择题。
- 科目二：系统集成项目管理应用技术（案例分析），机考问答题。
- 两科连续作答总时长240分钟。
- 科目一最短90分钟、最长120分钟，节余时间可用于科目二。
- 合格标准按每科试卷满分的60%计算；只有当管理员把单科满分配置为75分时，
  对应合格线才是45分。
- 题量、时长和分数始终从可编辑配置读取。
- 2026-07-18已核验2026年上半年官方批次安排：本资格为初、中级连考，
  两科总时长240分钟，基础知识最短90分钟、最长120分钟，节余时间可供
  应用技术使用；2026年上半年安排了三个批次，实际批次以准考证为准。
- 官方资格说明明确基础知识为机考选择题、应用技术（案例分析）为机考
  问答题；当前官方公开页面没有给出题量、单题分值或每科满分。
- Phase 5第一版只从已发布、无争议且可自动判分的原创客观题组卷。
  模板中的题量、题目权重、时长和满分均可编辑；不把未核实的题量或
  单题分值写死，也不伪装实现主观题自动评分。

## 2026-07-18：每日任务

- 默认每天学习1个知识点并完成20道题。
- 优先选择逾期、到期复习题，基础任务最多包含12道复习题。
- 普通练习按当前章节60%、薄弱知识点30%、未做随机题10%生成。
- 到期复习积压超过20题时，新知识点改为可选。
- 可选加练10题，不计入基础任务。

## 2026-07-18：知识掌握度

`掌握度 = 首次正确率×20% + 最近正确率×35% + 复习成功率×25% +
题目覆盖度×10% + 复习及时度×10%`。

- 未学习：没有阅读或作答记录。
- 学习中：不同题目少于3道，或有效作答少于5次。
- 薄弱：数据量达标后总分低于60，或最近正确率低于50%，或逾期超过7天。
- 基本掌握：数据量达标、总分不低于60、最近正确率不低于50%，且无严重逾期。

## 2026-07-18：AI

- Provider：DeepSeek。
- 模型ID：`deepseek-v4-pro`。
- 每日最多50次调用，每日估算费用硬上限人民币5元。
- 默认单次最多约12,000输入Token和3,000输出Token。
- 按API实际Token记录成本，北京时间零点重置。
- AI不可用或预算耗尽时，基础学习和人工解析必须继续可用。

## 2026-07-18：Phase 2内容存储

- 学习目录固定服务于系统集成项目管理工程师，不建设多考试入口。
- 本地开发未配置Supabase时使用进程内内容仓库，便于验证完整手机流程。
- 进程内数据不作为持久数据，开发服务器重启后恢复为内置原创示例。
- 生产环境未配置Supabase时失败关闭，不能自动降级到内存存储。
- 所有正式数据仍按固定 `owner` 的 `user_id` 隔离。
- 学习者只能看到 `published` 内容；草稿、待审核和归档内容不进入目录。
- 导入按 `external_id` 和递增 `version` 幂等处理，拒绝版本倒退。
- 数据库导入失败时回滚本次内容变更，并保留失败导入任务和错误报告。

## 2026-07-18：资料来源治理

- 官方网页只用于核验考试事实并保存URL，不复制页面原文、图片或附件。
- 官方大纲和教程建议通过正规渠道购买，系统只收录独立表达的个人笔记
  和原创总结。
- 第三方公开仓库和刷题网站只作为调研线索；公开可访问或仓库标注MIT
  不代表其中第三方教材、教辅和解析获得再发布授权。
- 未核验授权的第三方资料不下载、不抽取、不导入，版权状态保持
  `unknown`。
- Phase 3只能使用原创题目、可追溯的官方公开内容或有明确授权的题目。
- 详细来源登记和发布门禁见 `docs/CONTENT_SOURCE_AUDIT.md`。

## 2026-07-18：Phase 3题库与练习

- 第一批可自动判分题型为单选、多选和判断；不把主观题伪装成确定机器
  评分。
- 多选默认全对才得分，选项顺序不影响结果。
- 练习池只使用每个 `external_id` 的最新版本；若最新版本不是
  `published`，旧发布版本也不再进入新会话。
- 会话在开始时固定题目ID、版本和顺序；题目更新只新增版本，不改变旧
  会话及历史作答。
- “未做题”按用户和题目 `external_id` 判断，做过旧版本也视为已做。
- 收藏跟随题目 `external_id`，作答历史保留实际题目ID和版本。
- 未提交当前题目前，服务端不向客户端返回正确答案、解析或历史作答。
- 本地未配置Supabase时使用进程内练习仓库；生产环境继续失败关闭。
