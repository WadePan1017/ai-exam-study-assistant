# 内容来源与版权审计

核验日期：2026-07-18

本文件规定知识点、题目和考试配置可以参考哪些资料，以及在什么条件下
允许进入系统。它是项目内部的内容治理记录，不构成法律意见。

## 1. 当前结论

- 开发应用和核验考试基本规则不需要购买资料。
- 批量建设高质量知识内容前，建议合法购买当前官方考试大纲和教程。
- 官方网页只用于核验事实和保存来源链接，不复制网页原文、图片或附件。
- 用户购买教材后可以自行做个人笔记；系统只能收录重新组织、独立表达的
  原创总结，不收录教材扫描件或大段原文。
- `xiaomabenten/scpm_engineer` 只作为资料线索和功能调研入口，不直接
  导入其中的PDF、图片、真题解析或推荐网站题库。
- 来源或授权状态无法确认的内容保持 `unknown`，不得发布。

## 2. 来源分级

| 等级 | 典型来源 | 允许用途 | 入库要求 |
| --- | --- | --- | --- |
| A：官方事实 | 软考官网、人社部等政府部门页面 | 核验科目、时长、考试日期、合格比例等事实 | 使用自己的表述，保存页面URL与核验日期 |
| B：合法持有的参考书 | 正规渠道购买的官方大纲、教程 | 个人学习、整理原创知识总结 | 不上传书籍，不复制大段原文；标记为个人笔记或原创整理 |
| C：明确授权内容 | 作者或权利人提供书面许可的题目、解析 | 按授权范围使用 | 保存授权主体、范围、期限和证明 |
| D：第三方公开线索 | GitHub资料目录、博客、培训网站 | 发现主题、交叉检查遗漏、调研产品功能 | 不把“公开可访问”等同于“允许复制” |
| E：AI草稿 | DeepSeek生成的总结或练习题 | 辅助起草 | 标记 `ai_draft`，人工核对事实、答案和相似性后才能发布 |
| 禁止 | 盗版教材、商业题库抓取、来源不明解析 | 无 | 不下载、不导入、不改写规避版权 |

## 5. 内容入库写法

官方事实的来源说明示例：

```json
{
  "type": "official_public",
  "note": "SRC-OFFICIAL-2026-SCHEDULE；2026-07-18核验；仅依据公开事实原创总结",
  "copyright_status": "official_public"
}
```

根据自己合法持有的教材整理个人笔记：

```json
{
  "type": "user_note",
  "note": "个人根据正规购买的官方教程第3版整理；独立表达，未复制原文",
  "copyright_status": "user_note"
}
```

AI辅助起草：

```json
{
  "type": "ai_draft",
  "note": "DeepSeek辅助起草；等待人工核对事实、答案和相似性",
  "copyright_status": "ai_draft"
}
```

## 6. 发布门禁

每条知识点发布前必须确认：

1. `source.note` 能指向本文件中的来源编号，或说明合法持有/授权情况。
2. 正文是独立表达，不是教材、网页或解析的连续复制。
3. 涉及日期、法规、标准或考试规则时记录核验日期。
4. 来源为AI时已经人工核对，并将不确定内容删除或降为草稿。
5. `copyright_status` 不是 `unknown`。

每道题目发布前还必须确认：

1. 题干、选项和解析为原创或有可证明的授权。
2. 答案唯一，或案例题评分规则明确。
3. 没有通过替换数字、人物或选项顺序来伪装复制商业题目。
4. 标称“官方真题”时能够直接追溯到官方公开来源；否则只能标记为原创
   练习或来源待核验。

## 7. 后续执行顺序

1. 用户按需要购买官方大纲和教程，不上传盗版PDF。
2. 先为18个一级模块建立二、三级目录清单。
3. 每次只编写一个小批次原创知识点，先以 `draft` 导入。
4. 人工核对来源、事实和表达后改为 `reviewed`。
5. 只有通过本文件发布门禁的内容才能改为 `published`。
6. Phase 3题库开发仍需单独批准；批准后先制作少量原创示例题验证流程，
   不以第三方仓库作为题库种子。
