# 各阶段发给 Codex 的提示词

## Phase 0

```text
请阅读根目录AGENTS.md和docs/spec全部文件，执行Phase 0。
只审查仓库并输出：
1. 当前结构与技术栈；
2. 已有功能；
3. 与规格冲突；
4. 数据库和安全风险；
5. Phase 1至Phase 8实施计划；
6. Phase 1预计修改文件；
7. 需要我确认的问题。
本轮不要大规模编码。
```

## Phase 1

```text
我已确认Phase 0计划。现在只实施Phase 1：工程基础、登录和应用外壳。
严格遵守docs/spec/04_PHASE_PLAN_AND_ACCEPTANCE.md。
完成后运行lint、typecheck、unit tests和production build，输出：
- 变更摘要
- 文件清单
- 数据库迁移
- 测试结果
- 手动验收步骤
- 已知问题
不要进入Phase 2。
```

## Phase 2

```text
现在只实施Phase 2：大纲、知识点和内容导入。
必须包含18个一级模块种子数据、三级目录支持、知识点详情、个人笔记、JSON导入校验/预览/事务提交/报告。
先给出具体计划，再编码。完成后按规格验收，不进入Phase 3。
```

## Phase 3

```text
现在只实施Phase 3：题库和练习引擎。
实现题目导入、顺序/章节/随机/未做题练习、可靠保存、历史作答、客观题判分、解析、收藏和不确定标记。
不要实现错题复习算法和模拟考试，它们属于后续阶段。
```

## Phase 4

```text
现在只实施Phase 4：错题本和间隔复习。
实现错题自动归档、错误原因、1/3/7/14/30/60天算法、到期队列、逾期队列、再练和掌握状态。
重点为算法编写完整单元测试。
```

## Phase 5

```text
现在只实施Phase 5：模拟考试。
考试题量、时长、权重必须配置驱动。
实现倒计时、自动保存、刷新恢复、断网暂存、题号导航、交卷、超时自动提交、客观题评分和结果拆解。
模拟考试提交前不得显示答案或启用答案型AI。
```

## Phase 6

```text
现在只实施Phase 6：首页和学习报告。
所有统计必须来自真实记录，不制造虚假能力值。
优化大量作答记录下的查询性能，并为连续学习天数、薄弱章节和报告汇总写测试。
```

## Phase 7

```text
现在只实施Phase 7：AI讲题。
先建立provider适配层和服务端安全调用，再实现题目解释、选项分析、计算步骤、概念对比和变式题草稿。
不得把AI内容自动发布，不得暴露密钥。
没有AI配置时核心学习功能必须正常运行。
```

## Phase 8

```text
现在只实施Phase 8：PWA、备份、发布和加固。
完成数据导出/恢复、PWA安装、离线降级、CI、可访问性、安全审查、性能优化、部署与回滚文档。
执行完整回归测试并给出最终验收报告。
```
