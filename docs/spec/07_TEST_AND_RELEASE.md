# 测试、发布和回滚

## 1. 必测业务逻辑

单元测试：

- 客观题判分
- 多选题规则
- 复习间隔算法
- 连续学习天数
- 章节掌握度
- 模考deadline
- 模考分数汇总
- 导入校验
- 内容版本比较
- 备份格式校验

集成测试：

- 作答写入attempt并更新状态
- 错题进入复习
- 模考提交生成报告
- 导入事务成功/失败回滚
- RLS权限
- 数据导出和恢复

E2E：

1. 登录
2. 打开知识点
3. 开始章节练习
4. 答错一道题
5. 在错题本看到该题
6. 完成复习
7. 开始并提交模拟考试
8. 查看报告
9. 管理员导入题目
10. 导出学习数据

## 2. 每阶段检查命令

根据项目实际脚本调整，但必须包含：

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:e2e
pnpm build
```

不得通过关闭类型检查或跳过测试来制造“通过”。

## 3. 数据库迁移

- 每次结构变化创建迁移
- 迁移可在空数据库执行
- 重要破坏性迁移提供备份和回滚说明
- 不直接手改生产数据库
- seed数据不覆盖用户数据

## 4. 发布环境

至少区分：

- local
- preview/staging
- production

环境变量分别配置。预览环境不能使用生产service role key。

## 5. 回滚

每阶段：

- 使用独立分支或清晰提交
- 记录迁移编号
- 记录部署版本
- 写明回滚代码步骤
- 写明数据库回滚或向前修复策略

涉及用户数据的迁移，优先向前修复，不轻率回滚并丢数据。

## 6. 发布检查表

- lint通过
- typecheck通过
- 单测通过
- E2E通过
- build通过
- migration已验证
- `.env.example`更新
- README更新
- 无密钥提交
- 手机端核心流程测试
- 数据导出测试
- 错误监控可用
- 备份完成
