# UI / UX规格

## 1. 设计原则

- 移动端优先
- 中文界面
- 信息密度适中
- 重点清晰，不做儿童化游戏界面
- 一只手可完成大部分刷题操作
- 所有关键操作都有反馈
- 不用颜色作为唯一状态提示

## 2. 视觉建议

- 主色稳重、偏科技与学习场景
- 背景和卡片层次清晰
- 正确、错误、警告颜色遵循可访问性对比
- 字号不低于移动端可读标准
- 长篇知识内容有目录、段落和公式块
- 支持浅色和深色模式，可放在后期完成

- 产品名称

- 最近模考

- 进入知识点

## 4. 状态设计

每页都必须覆盖：

- loading
- empty
- error
- offline
- permission denied
- partial data

示例：

- 没有错题时，不显示空白页面，显示“当前没有错题”和进入章节练习按钮
- 导入失败时，展示具体行号和字段
- AI不可用时，人工解析仍可正常查看

## 5. 可访问性

- 所有按钮有可读名称
- 键盘可操作
- 焦点状态清晰
- 表单错误与字段关联
- 弹窗正确管理焦点
- 题目选项不只用颜色显示对错
- 图表提供文本摘要
